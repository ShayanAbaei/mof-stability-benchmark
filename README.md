\# Project 15 — MOF stability benchmark



This repository accompanies the manuscript:



\*\*Interpretable Metal-Chemistry and Minimal-Context Descriptors for MOF Water and Activation Stability: A Reproducible Benchmark of Descriptor Sufficiency, Split Sensitivity, and Screening Utility\*\*



\## Repository purpose



This repository provides the code notebooks and paper files used for the Project 15 MOF stability benchmark.



The repository is intentionally lightweight. It contains the workflow notebooks, manuscript files, Supporting Information files, and documentation needed to understand and rerun the benchmark.



\## What is included



\- main benchmark workflow notebook

\- result-generation / figure-table builder notebook

\- final manuscript files

\- final Supporting Information files

\- workflow and output documentation

\- data provenance note

\- citation and environment files



\## What is not included



This repository does \*\*not\*\* redistribute the raw external database files or the generated benchmark output folders.



The raw input data were obtained from the public CoRE MOF 2024 Zenodo dataset release and must be downloaded separately.



The generated outputs of the notebooks — including pickled results objects, checkpoints, logs, manuscript-ready figures, tables, and run summaries — are also not stored in this repository. They can be regenerated locally by running the workflow.



\## Required raw input files



The main workflow expects the following files inside a local folder named `raw\_data/`:



\- `ASR\_data\_SI\_20250204.csv`

\- `FSR\_data\_SI\_20250204.csv`

\- `ION\_data\_SI\_20250204.csv`

\- `ASR\_FSR\_check.csv`

\- `12089-recommended-screening-list.csv`



These files were obtained from the public CoRE MOF 2024 dataset release on Zenodo.



\## Workflow summary



1\. Download the required raw CSV files from the external Zenodo source.

2\. Place them in a local folder named `raw\_data/`.

3\. Run `project\_stability\_benchmark\_colab\_Workflow\_Main.ipynb`.

4\. That notebook generates the benchmark results object and the main intermediate outputs needed for downstream processing.

5\. Run `project\_figure\_table\_builder\_revised\_colab\_Result\_Generation\_Cleaner.ipynb`.

6\. That notebook reads the results object generated by the main workflow and exports cleaned figures, tables, logs, checkpoints, and summary files for manuscript preparation.



\## Notes on outputs



The repository does not store generated outputs directly. Instead, the expected outputs and their roles are documented in `docs/WORKFLOW\_AND\_OUTPUTS.md`.



Some final manuscript and Supporting Information figures may include minor publication-oriented post-processing outside the notebooks.
