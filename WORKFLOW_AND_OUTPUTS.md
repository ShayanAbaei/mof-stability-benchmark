# Workflow and expected outputs

This document explains what each notebook does and what outputs it generates.

## Notebook 1
`project_stability_benchmark_colab_Workflow_Main.ipynb`

### Purpose
This is the main benchmark workflow.

### Input
It reads the five raw CSV files stored in a local folder named `raw_data/`:
- ASR_data_SI_20250204.csv
- FSR_data_SI_20250204.csv
- ION_data_SI_20250204.csv
- ASR_FSR_check.csv
- 12089-recommended-screening-list.csv

### Main operations
This notebook performs:
- raw table loading
- audit and cleaning
- identifier normalization
- engineered variable construction
- descriptor-regime definition
- regression benchmark generation
- thresholded classification benchmark generation
- transfer analysis
- subgroup analysis
- failure analysis
- export of the benchmark results object

### Main generated artifact
The main workflow generates a serialized results object, typically stored as:

`project_master_results_latest.pkl`

This object is then used by the second notebook.

## Notebook 2
`project_figure_table_builder_revised_colab_Result_Generation_Cleaner.ipynb`

### Purpose
This notebook reads the results object generated by the main workflow and produces cleaned outputs for manuscript preparation.

### Input
- `project_master_results_latest.pkl`

### Main operations
This notebook performs:
- result aggregation
- benchmark summary extraction
- figure generation
- table export
- log writing
- run summary export
- cleanup and final formatting of output bundles

### Typical generated outputs
This notebook can generate a local output directory containing items such as:
- `checkpoints/`
- `figures/`
- `tables/`
- `logs/`
- `run_summary.json`

### Role of these outputs
- `figures/` contains manuscript and Supporting Information figure exports
- `tables/` contains benchmark summary tables and support tables
- `logs/` records workflow progress and export events
- `run_summary.json` stores a compact summary of the completed run
- `checkpoints/` stores intermediate workflow snapshots or convenience outputs

These generated outputs are not stored in this repository, but they can be recreated locally by rerunning the notebooks.
